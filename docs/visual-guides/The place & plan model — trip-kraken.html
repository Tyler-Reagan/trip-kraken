Data model · trip-kraken

One place type, one plan

A trip is a set of typed Locations plus a plan that arranges them on a date axis. Lodging, transit, and activity are kinds of one primitive; the temporal facts about a place are fields on it; the plan is the only thing the optimizer produces.

The whole model in one breath

One primitive

Location, typed by kind

Every place is one Location in one table, discriminated by kind: activity · transit · lodging. Functions over Location inherit to all of them.

One seam

Constraint vs. plan

Facts about a place (a stay’s dates, a transit’s times) are fields — optimizer inputs. The arrangement of places is the Plan — the only output.

One axis

Real dates, always

Every trip has a date range; day-numbers derive from it. A place’s presence across days is projected from its fields, never stored twice.

The taxonomy: one interface, extended by kind

A discriminated union over a single table. Subtype interfaces are added lazily — only when a kind (or a leaf) carries fields of its own. A value stays at the nearest meaningful level until it earns a deeper one.

Functions-over-records, not methods

A hotel is a Location; f(loc) works on it for free. Narrow on kind only for the rare lodging-specific bit. No runtime classes, no hydration.

Depth is data, not types

Restaurant vs. retail vs. airport already lives in the Places categories string. Mint a subtype interface only when it adds a field.

The seam: constraint in, plan out

The Manifest is just Locations and their data. The Plan carries the weight of contextualizing them onto a timeline — clustered by day. It does so two ways: it projects each constraint-field across the days it covers, and it places the activities.

The invariant

Entities are nouns; roles are adjectives. A place is never a lodging — it’s a Location of kind: lodging.

Gone for good: the Stay entity (its dates fold onto Lodging) · locking · same-place multiplicity · the date / day-number dual mode · stored roles & isLodging · stored trip edges (derived from earliest/latest transit).
